# 产品体验优化开发计划

> 来源文档：`docs/product-experience-review.md`  
> 计划日期：2026-05-07  
> 目标：把体验评估中的建议拆成可开发、可自测、可验收、可分阶段提交的任务。

## 开发原则

1. 每个小功能完成后做局部自测，并记录到 `docs/progress.md`。
2. 每个大功能完成后做整体自测、提交 Git，并推送到 GitHub。
3. 遇到业务矛盾或配置口径不清时暂停确认，不自行扩大范围。
4. 优先修复影响注册、付费、生成成功感和复用效率的问题。
5. 本计划先不引入复杂用户分组、团队空间、多版本协作等重模型能力。

## 总体优先级

### P0：影响注册、付费和核心体验

1. 首页游客规则、注册收益和场景价值表达。
2. 免费额度耗尽后的友好转化引导。
3. 套餐页计费规则和可生成张数说明。
4. 历史记录支持复制 prompt、再次生成。
5. 后台设置分组和敏感配置保护。

### P1：提升创作效率和留存

1. 场景化 prompt 模板。
2. 尺寸选项绑定平台用途。
3. 历史搜索、筛选、失败重试。
4. 结果区参数快照和继续编辑入口。
5. 用户菜单聚合历史、积分、套餐入口。

### P2：增强运营和风控能力

1. 后台概览增加失败率、渠道可用率、积分消耗趋势。
2. 渠道测试记录最近结果和错误原因。
3. 管理员操作审计。
4. 积分流水筛选。
5. 告警策略可视化配置。

## Milestone 1：首页转化与生成前引导

### 目标

让第一次访问的用户快速知道“能做什么、免费规则是什么、注册后有什么收益”，减少生成前的不确定感。

### 小功能拆分

#### 1.1 首页价值表达和游客规则提示

- 开发范围：
  - 调整首页主标题、副标题和生成框附近的权益提示。
  - 展示游客免费次数、新用户注册赠送积分、历史保存收益。
  - 不改变现有生成 API 和参数逻辑。
- 局部验收：
  - 游客首页能直接看到免费规则。
  - 登录用户不显示游客免费次数误导文案。
  - 页面仍以生图输入区为第一操作焦点。
- 局部自测：
  - `pnpm.cmd exec vue-tsc --noEmit`
  - 浏览器检查游客态、登录态文案差异。
- 文档更新：
  - 在 `docs/progress.md` 记录首页文案和游客规则提示已完成。

#### 1.2 场景化 prompt 模板后端扩展

- 开发范围：
  - 复用 `prompt_templates`，新增 `category=scenario`。
  - 增加默认场景模板数据，如小红书封面、商品展示图、头像、海报、壁纸。
  - 保持后台模板管理可编辑、可启停。
- 局部验收：
  - `/api/prompt-templates` 返回 `scenario` 分类模板。
  - 后台模板列表能识别并显示场景模板分类。
  - 关闭某个场景模板后前台不展示。
- 局部自测：
  - `go test ./controller -run "TestPromptTemplate" -v`
  - `go test ./controller -run "TestAdminPromptTemplate" -v`
- 文档更新：
  - 在 `docs/progress.md` 记录模板分类扩展和默认场景。

#### 1.3 首页场景入口 UI 与参数联动

- 开发范围：
  - 首页展示 3 到 5 个场景入口。
  - 点击场景入口后带入 prompt，并推荐尺寸和风格。
  - 如果后端模板加载失败，保留兜底场景，不阻塞生成。
- 局部验收：
  - 点击场景入口后输入框、尺寸、风格同步变化。
  - 用户仍可手动修改带入后的内容。
  - 场景入口不会挤压主要生成按钮。
- 局部自测：
  - `pnpm.cmd exec vue-tsc --noEmit`
  - 浏览器手动点击每个场景入口验证联动。
- 文档更新：
  - 在 `docs/progress.md` 记录场景入口和联动规则。

#### 1.4 免费额度耗尽提示优化

- 开发范围：
  - 优化 `CreditExhaustedGuide` 文案和按钮。
  - 对游客显示注册收益，对登录用户显示充值/联系客服引导。
  - 确保不再出现 `free trial used, please register` 这类英文接口文案。
- 局部验收：
  - 游客额度用完后看到中文友好提示。
  - 登录用户积分不足时看到充值或联系引导。
  - 提示不会重复出现在多个位置。
- 局部自测：
  - `pnpm.cmd exec vue-tsc --noEmit`
  - 手动模拟游客额度耗尽和登录用户积分不足。
- 文档更新：
  - 在 `docs/progress.md` 记录额度耗尽提示改造。

#### 1.5 登录/注册后保留生成参数

- 开发范围：
  - 游客跳转登录/注册前缓存 prompt、尺寸、风格、模式。
  - 登录/注册成功回到首页后恢复这些参数。
  - 缓存只保存非敏感生成参数，不保存 token 或密钥。
- 局部验收：
  - 游客输入内容后去登录，回来仍能看到原内容。
  - 缓存过期或手动清空时不影响页面打开。
  - 管理员登录不错误带回普通生图页。
- 局部自测：
  - `pnpm.cmd exec vue-tsc --noEmit`
  - 手动走游客到登录再返回首页流程。
- 文档更新：
  - 在 `docs/progress.md` 记录参数保留逻辑和限制。

### 大功能整体验收与提交

- 整体验收：
  - 游客首页价值表达、场景入口、额度耗尽引导、登录后参数恢复串联可用。
  - 登录用户、游客、管理员三种身份入口不混淆。
- 整体自测：
  - `go test ./controller -run "TestPromptTemplate" -v`
  - `go test ./...`
  - `pnpm.cmd exec vue-tsc --noEmit`
  - `pnpm.cmd build`
- 提交规则：
  - 通过后提交并推送，建议提交信息：`feat: improve home conversion guidance`

   - 在输入区附近增加一句轻量权益提示：游客免费次数、注册赠送积分、保存历史。
   - 避免营销落地页化，仍保持当前生图工具为首屏主体。

2. 场景入口
   - 在首页增加 3 到 5 个场景入口：小红书封面、商品展示图、头像、海报、壁纸。
   - 点击场景入口后自动填入对应 prompt 模板，并联动推荐尺寸和风格。
   - 场景模板优先复用现有 `prompt_templates`，新增 `category=scenario`。

3. 免费额度耗尽引导
   - 优化 `CreditExhaustedGuide` 文案。
   - 展示注册收益：新用户赠送积分、保存历史、可继续生成。
   - 保留用户当前填写的 prompt、尺寸、风格，登录/注册后返回首页继续使用。

- 游客打开首页能看到免费规则和注册收益。

- 游客额度用完后提示为中文友好表达，不再出现英文接口文案。
- 未登录用户填写的生成参数在跳转登录/注册后不丢失。

## Milestone 2：计费透明和套餐页优化

### 目标

让用户在生成前和购买前都能理解积分怎么扣、失败是否退回、套餐大约能生成多少张图。

### 小功能拆分

#### 2.1 抽取统一计费展示口径

- 开发范围：
  - 梳理当前尺寸计费逻辑，以后端实际扣费逻辑为准。
  - 前端统一使用同一套尺寸消耗计算或后端返回值。
  - 明确 1024x1024 为 1 积分，1536x1024 为 2 积分，其他尺寸按像素向上取整。
- 局部验收：
  - 首页显示消耗和后端创建任务扣费一致。
  - 尺寸切换后预计积分立即更新。
  - 不再出现前端显示 1 积分、后端实际 2 积分的问题。
- 局部自测：
  - `go test ./service -run "Test" -v`
  - `pnpm.cmd exec vue-tsc --noEmit`
- 文档更新：
  - 在 `docs/progress.md` 记录计费口径。

#### 2.2 首页计费规则入口

- 开发范围：
  - 首页预计消耗区域增加“查看计费规则”入口。
  - 弹窗或抽屉展示尺寸、积分、失败退回说明。
  - 说明文案使用用户语言，不暴露内部状态码。
- 局部验收：
  - 用户点击后能看懂不同尺寸为什么消耗不同。
  - 积分不足时能从提示进入套餐页。
  - 弹窗关闭后不影响当前 prompt 和参数。
- 局部自测：
  - `pnpm.cmd exec vue-tsc --noEmit`
  - 浏览器检查尺寸切换和计费说明。
- 文档更新：
  - 在 `docs/progress.md` 记录首页计费说明入口。

#### 2.3 套餐卡片可生成张数

- 开发范围：
  - 套餐页每个套餐显示“约可生成 N 张标准图”。
  - 标准图以 1024x1024、1 积分为基准。
  - 如果套餐赠送积分或有效期存在，也纳入展示说明。
- 局部验收：
  - 套餐积分和可生成张数换算正确。
  - 套餐为空或接口失败时有兜底提示。
  - 不影响原购买入口。
- 局部自测：
  - `pnpm.cmd exec vue-tsc --noEmit`
  - 手动检查套餐页展示。
- 文档更新：
  - 在 `docs/progress.md` 记录套餐页转化信息增强。

#### 2.4 积分规则说明与流水入口

- 开发范围：
  - 套餐页增加规则说明：消耗规则、有效期、失败退回。
  - 增加普通用户积分流水入口。
  - 如已有积分流水接口不足，先补用户侧只读接口。
- 局部验收：
  - 用户能从套餐页进入自己的积分流水。
  - 用户只能看到自己的流水。
  - 管理员积分流水不和用户侧入口混淆。
- 局部自测：
  - `go test ./controller -run "TestCredit" -v`
  - `pnpm.cmd exec vue-tsc --noEmit`
- 文档更新：
  - 在 `docs/progress.md` 记录积分流水入口和权限规则。

#### 2.5 支付失败/生成失败扣费规则核对

- 开发范围：
  - 核对生成失败、取消、上传失败时积分是否退回。
  - 核对支付失败是否不会增加积分。
  - 如实际逻辑和展示不一致，优先修正展示；涉及业务规则变化时先确认。
- 局部验收：
  - 文案和代码行为一致。
  - 测试覆盖失败不错误扣费或退款行为。
- 局部自测：
  - `go test ./service ./controller -run "Test.*Credit|Test.*Generation" -v`
- 文档更新：
  - 在 `docs/progress.md` 记录失败扣费核对结果。

### 大功能整体验收与提交

- 整体验收：
  - 首页预计消耗、套餐页可生成张数、积分流水、失败规则说明口径一致。

  - `go test ./...`
  - `pnpm.cmd exec vue-tsc --noEmit`
  - `pnpm.cmd build`
- 提交规则：
  - 通过后提交并推送，建议提交信息：`feat: clarify credit billing experience`

### 开发内容

1. 计费规则统一展示
   - 首页预计消耗区域增加“查看计费规则”入口。
   - 计费规则按当前尺寸像素向上取整逻辑展示。
   - 明确失败任务不扣费或失败后退回积分的当前系统规则，以实际代码为准。

2. 套餐页增强
   - 套餐卡片增加“约可生成 N 张标准图”。
   - 套餐页增加积分说明区：消耗规则、有效期、失败/退款说明。
   - 增加积分流水入口。

3. 后台可配置说明
   - 如当前说明文案需要运营调整，增加后台设置项。
   - 不把支付密钥、渠道密钥等敏感字段暴露到前端。

## Milestone 3：历史记录改造成作品资产库第一阶段

### 目标

让老用户和专业用户能复用已有作品，减少重复输入和重复配置。

### 小功能拆分

#### 3.1 历史列表搜索和筛选接口

- 开发范围：
  - 历史接口支持 prompt 关键词、状态、尺寸筛选。
  - 保持分页能力。
  - 只返回当前用户自己的记录。
- 局部验收：
  - 搜索关键词能匹配 prompt。
  - 状态和尺寸筛选能组合使用。
  - 越权访问其他用户记录失败。
- 局部自测：
  - `go test ./controller -run "TestHistory" -v`
- 文档更新：
  - 在 `docs/progress.md` 记录历史筛选接口。

#### 3.2 历史页筛选 UI

- 开发范围：
  - 历史页增加关键词搜索框、状态筛选、尺寸筛选。
  - 查询条件变化后刷新列表。
  - 空结果显示友好空态。
- 局部验收：
  - 搜索/筛选/清空筛选可用。
  - 移动端不挤压历史卡片。
  - 原下载和删除功能不受影响。
- 局部自测：
  - `pnpm.cmd exec vue-tsc --noEmit`
  - 浏览器手动验证筛选组合。
- 文档更新：
  - 在 `docs/progress.md` 记录历史筛选 UI。

#### 3.3 复制 prompt

- 开发范围：
  - 历史卡片增加复制 prompt 按钮。
  - 复制成功/失败显示轻提示。
  - 对空 prompt 或异常记录做兜底。
- 局部验收：
  - 成功复制完整 prompt。
  - 浏览器不支持剪贴板时有降级提示。
  - 按钮位置不影响下载/删除。
- 局部自测：
  - `pnpm.cmd exec vue-tsc --noEmit`

  - 在 `docs/progress.md` 记录复制 prompt 能力。

  - 历史卡片增加“再次生成”。
  - 点击后把 prompt、尺寸等参数带回首页，由用户手动确认生成。
  - 不自动扣费、不自动创建任务。
- 局部验收：
  - 首页能恢复历史 prompt 和尺寸。
  - 用户可以继续修改后再生成。
  - 失败记录也可以带回首页重试。
- 局部自测：

  - 手动从历史跳转首页验证参数。
- 文档更新：
  - 在 `docs/progress.md` 记录再次生成交互。

#### 3.5 失败记录原因和重试入口

- 开发范围：
  - 历史页失败记录展示错误摘要。
  - 错误摘要做用户友好改写。

- 局部验收：
  - 失败记录不只显示失败状态。
  - 用户能看到下一步动作。
  - 不暴露过长上游错误和敏感 URL/API Key。
- 局部自测：
  - `go test ./controller -run "TestHistory" -v`

- 文档更新：
  - 在 `docs/progress.md` 记录失败记录展示策略。

## 需要确认的问题

1. 场景入口首批是否固定为：小红书封面、商品展示图、头像、海报、壁纸？
