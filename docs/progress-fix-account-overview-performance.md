# 个人中心加载慢修复记录

## 背景

用户反馈进入个人中心加载很慢。

排查发现 `/api/account/overview` 在生成统计上存在多次串行查询：

- 累计生成数 `COUNT`
- 完成数 `COUNT`
- 失败数 `COUNT`
- 最新生成时间查询
- 最近作品列表查询

个人中心首屏还会等待完整概览返回后再展示主体内容，导致用户体感更慢。

## 修复拆分

### 小功能 1：优化账号概览生成统计查询

状态：已完成

改动：

- 将生成统计的多次串行查询合并为一次聚合查询。
- 保持 `total`、`completed`、`failed`、`latest_at`、`recent_items` 返回结构不变。
- 最近作品预览查询只选择前端需要展示的字段，减少数据读取量。
- 自测发现 SQLite 对 `MAX(created_at)` 扫描到 `*time.Time` 有兼容问题，已改为用最近作品第一条的 `created_at` 作为 `latest_at`，避免不同 SQLite 驱动时间类型差异。

自测：

- 第一次执行 `go test ./controller -run "TestAccountOverview" -v`：失败，原因是 SQLite 聚合时间字段扫描兼容问题。
- 复测 `go test ./controller -run "TestAccountOverview" -v`：通过。

### 小功能 2：优化个人中心首屏加载体验

状态：已完成

改动：

- 当前端已有登录用户缓存时，先展示用户信息和个人资料表单。
- 概览接口继续后台刷新统计、通知、最近登录等信息。
- 无用户缓存时仍展示加载态，避免空数据误导。

自测：

- `pnpm.cmd exec vue-tsc --noEmit`：通过。
