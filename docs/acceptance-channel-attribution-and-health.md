# 渠道归因与渠道健康统计自测验收文档

> 计划日期：2026-05-07  
> 对应开发计划：`docs/plan-channel-attribution-and-health.md`

## 验收目标

确认系统能在生成任务中记录实际使用渠道，并在后台渠道页展示近 24 小时成功、失败和失败率，同时不影响现有生成、渠道测试和历史数据。

## 小功能自测清单

### 1.1 生成记录渠道字段扩展

测试点：

- `generations` 自动迁移新增字段成功。
- 旧数据 `channel_id/channel_name` 为空时仍可查询历史。
- JSON 返回不会破坏现有前端字段。

命令：

```powershell
go test ./model -v
go test ./... 
```

通过标准：

- 测试全部通过。
- 无迁移错误。

### 1.2 生成渠道选择结果回传

测试点：

- 单渠道成功时返回该渠道 ID 和名称。
- 多渠道中第一个失败、第二个成功时，返回第二个渠道。
- 环境变量兜底渠道返回固定名称。
- 上游全部失败时，返回最后尝试渠道。

命令：

```powershell
go test ./service -run "Test.*Channel|Test.*GenerateImage" -v
```

通过标准：

- 成功和失败路径都能拿到符合口径的渠道信息。
- 原有多渠道重试测试不回归。

### 1.3 生成任务写入渠道归因

测试点：

- 成功生成后，数据库记录有渠道归因。
- 上游最终失败后，数据库记录有最后尝试渠道。
- 保存图片失败后，生成记录仍保留上游渠道。
- 待处理状态取消不误写渠道。

命令：

```powershell
go test ./service -run "Test.*Generation|Test.*Channel" -v
go test ./controller -run "TestGeneration" -v
```

通过标准：

- `channel_id/channel_name` 写入符合业务口径。
- 失败退款、取消逻辑不回归。

### 1.4 渠道近 24 小时统计接口

测试点：

- 同一渠道 24 小时内成功/失败数量正确。
- 24 小时外记录不计入。
- 无记录渠道返回 0。
- 禁用渠道仍展示历史近 24 小时统计。
- 老数据无渠道归属时不污染任一渠道统计。

命令：

```powershell
go test ./controller -run "TestAdminChannel" -v
```

通过标准：

- 接口返回计数和失败率正确。
- 后台渠道 CRUD、渠道测试功能不回归。

### 1.5 后台渠道页展示统计

测试点：

- 渠道列表显示近 24 小时成功数、失败数、失败率。
- 无统计数据时显示清晰空状态。
- 最近测试结果与生成统计分别展示，不混淆。
- 窄屏下统计标签不遮挡测试、编辑、删除按钮。

命令：

```powershell
pnpm.cmd exec vue-tsc --noEmit
```

人工验收：

- 管理后台进入“渠道”页。
- 检查有统计数据、无统计数据、测试失败渠道三种状态。

通过标准：

- 页面无类型错误。
- 统计显示清晰，不影响原有操作。

## 整体验收清单

### 后端整体

命令：

```powershell
go test ./service -run "Test.*Generation|Test.*Channel" -v
go test ./controller -run "TestAdminChannel|TestGeneration" -v
go test ./...
```

通过标准：

- 所有测试通过。
- 生成、退款、取消、渠道测试、后台渠道列表不回归。

### 前端整体

命令：

```powershell
pnpm.cmd exec vue-tsc --noEmit
pnpm.cmd build
```

通过标准：

- 类型检查通过。
- 生产构建通过。
- 如果沙箱内 `pnpm.cmd build` 因 esbuild `spawn EPERM` 失败，允许按既有流程提权重跑，并记录到 `docs/progress.md`。

### 人工验收

准备数据：

- 至少 1 个启用渠道。
- 至少 1 个成功生成任务。
- 至少 1 个失败生成任务。
- 至少 1 条 24 小时外旧记录。

验收步骤：

1. 创建或使用已有渠道，执行一次成功生成。
2. 模拟上游失败，执行一次失败生成。
3. 打开后台 `/console/admin`。
4. 进入“渠道”页。
5. 检查该渠道近 24 小时成功数、失败数、失败率。
6. 检查最近测试结果仍可正常展示。
7. 打开“监控”页，确认整体失败率和最近失败任务仍正常。

通过标准：

- 渠道页统计与数据库记录可核对。
- 后台不出现空白页或控制台错误。
- 旧生成记录不会被错误归入某个渠道。

## 发布前 Go/No-Go 标准

Go：

- 所有自动化测试通过。
- 生产构建通过。
- 渠道统计和最近测试结果展示不混淆。
- 文档已更新进度和已知限制。

Conditional Go：

- 仅人工验收中的展示样式有轻微问题，但不影响统计准确性。

No-Go：

- 生成任务无法稳定写入渠道归因。
- 渠道统计计数错误。
- 生成、退款、取消任一核心流程回归。
