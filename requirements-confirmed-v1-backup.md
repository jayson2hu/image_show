# GPT-Image-2 图片生成网站 — 需求确认与开发计划

> 确认日期：2026-04-29
> 状态：需求确认阶段

---

## 一、已确认需求摘要

### 用户画像
- **目标用户**：国内用户为主
- **开发者身份**：个人开发者，无企业资质
- **服务器资源**：日本腾讯轻量云 2C4G 30Mbps（已有其他服务在跑，资源有限）

### 技术栈
- **后端**：Go 语言
- **前端**：Vue（具体框架待定，Vue 3 + Vite 为主流选择）
- **部署**：后端部署在日本服务器，前端也部署到服务器，数据库使用new-api和sub2api同一个数据库

### 核心功能（MVP）
- 仅支持**文生图**，不含图片编辑
- 用户可调节参数：prompt、尺寸等（由管理员控制可选范围）
- 管理员后台控制可用参数和开关

## 二、四大问题决策确认

### Q1：费用与积分

| 决策项 | 确认结果 |
|-------|---------|
| 积分单位 | 1 积分 = 1 次标准生成（medium 1024×1024） |
| 不同质量消耗 | low 0.2 积分 / medium 1 积分 / high 4 积分 |
| 注册赠送 | 3 积分（7 天有效期） |
| 支付功能 | **暂不接入**，后端预留积分购买接口，前端暂不开放入口 |
| 积分充值 | MVP 阶段由管理员后台手动充值 |

### Q2：注册与登录

| 决策项 | 确认结果 |
|-------|---------|
| 注册方式 | **微信登录 + 邮箱注册**，两种并存 |
| 注册开关 | 管理员可控制是否开放注册（参考 new-api 模式） |
| SMTP 配置 | 支持管理员配置 SMTP，用于邮箱验证和通知 |
| 日志记录 | 记录登录日志（时间、IP、设备、登录方式），对应的使用日志，需要提供删除的接口，如果日志太多了 |
| 微信登录 | 个人开发者无企业资质，走**微信公众号测试号**或**第三方微信登录服务**（如 WxPusher 扫码关注登录）微信登录参考new-api |
| 安全加固 | 后续迭代加入，MVP 先做基础防护 |

### Q3：防刷策略

| 优先级 | 措施 | MVP 是否实现 |
|-------|------|-------------|
| P0 | OpenAI 账户设置 hard limit（如 $50/月） | 是（OpenAI 后台配置） |
| P0 | 用户级频率限制（10 次/分钟） | 是 |
| P0 | IP 级频率限制（Cloudflare WAF 或代码层） | 是，可以通过同一个ip限定次数 |
| P0 | 积分耗尽即停（天然硬限制） | 是 |
| P1 | 每日 API 支出预算 + 熔断 | 是（代码层 daily budget） |
| P1 | 管理员手动封禁用户 | 是 |
| P2 | 行为验证码 | 后续迭代 |
| P2 | Prompt 相似度检测 | 后续迭代 |

### Q4：图片存储与架构

| 决策项 | 确认结果 |
|-------|---------|
| 图片存储 | Cloudflare R2（零出口费用） |
| 图片分发 | 通过 Cloudflare CDN，不走服务器带宽 |
| 图片保留 | 免费用户 7 天 / 付费用户 90 天（后续开放） |
| 图片链接 | Presigned URL，防盗链 |
| 服务器角色 | 仅做 API 网关 + 业务逻辑，不承担静态资源分发 |

## 三、系统架构总览

```
┌─────────────┐     ┌──────────────────────┐     ┌─────────────┐
│  Vue 前端    │────→│  Cloudflare Pages    │     │  Cloudflare │
│  (SPA)      │     │  (静态托管，免费)      │     │  R2 + CDN   │
└──────┬──────┘     └──────────────────────┘     │  (图片存储)  │
       │ API 请求                                  └──────┬──────┘
       ▼                                                  ▲
┌──────────────────────────────────────┐                  │
│  日本服务器 2C4G                      │     上传图片      │
│  ┌────────────────────────────────┐  │──────────────────┘
│  │  Go 后端 (Gin/Fiber)           │  │
│  │  ├─ 用户认证 (JWT)             │  │
│  │  ├─ 积分管理                   │  │
│  │  ├─ OpenAI API 调用            │  │
│  │  ├─ 频率限制                   │  │
│  │  └─ 管理员后台 API             │  │
│  ├────────────────────────────────┤  │
│  │  SQLite (数据库)               │  │
│  │  ├─ 用户表                     │  │
│  │  ├─ 积分流水表                 │  │
│  │  ├─ 生成记录表                 │  │
│  │  └─ 登录日志表                 │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

**为什么选 SQLite 而不是 MySQL/PostgreSQL：**
- 服务器 2C4G 且有其他服务，SQLite 零额外内存开销
- MVP 阶段数据量小，SQLite 完全够用
- 后续如果需要可平滑迁移到 PostgreSQL

## 四、开发阶段划分

### Phase 1：基础骨架（P0，预计 3-5 天）

| 序号 | 任务 | 说明 |
|------|------|------|
| 1.1 | Go 后端项目初始化 | Gin/Fiber 框架，项目结构，配置管理 |
| 1.2 | 数据库设计 | 用户、积分、生成记录、登录日志表 |
| 1.3 | 邮箱注册/登录 | SMTP 配置、邮箱验证码、JWT 签发 |
| 1.4 | 注册开关 | 管理员可开关注册功能 |
| 1.5 | Vue 前端项目初始化 | Vue 3 + Vite + 路由 + 基础布局 |

### Phase 2：核心生成功能（P0，预计 3-5 天）

| 序号 | 任务 | 说明 |
|------|------|------|
| 2.1 | OpenAI API 对接 | gpt-image-2 文生图调用，sse支持 |
| 2.2 | 参数管理 | 管理员配置可用尺寸、质量档位 |
| 2.3 | 积分系统 | 扣减、查询、管理员充值 |
| 2.4 | 图片上传 R2 | 生成后上传 Cloudflare R2，返回 CDN URL |
| 2.5 | 生成页面 | prompt 输入、参数选择、生成结果展示 |

### Phase 3：防护与管理（P0-P1，预计 2-3 天）

| 序号 | 任务 | 说明 |
|------|------|------|
| 3.1 | 频率限制 | 用户级 + IP 级限流（Go 中间件） |
| 3.2 | 每日预算熔断 | API 支出超限自动暂停 |
| 3.3 | 管理员后台 | 用户管理、封禁、积分充值、参数配置 |
| 3.4 | 登录日志 | 记录 IP、设备、时间 |

### Phase 4：体验优化（P1，预计 2-3 天）

| 序号 | 任务 | 说明 |
|------|------|------|
| 4.1 | 微信登录 | 接入第三方微信登录方案 |
| 4.2 | 图片历史 | 用户查看自己的生成历史 |
| 4.3 | 图片生命周期 | R2 lifecycle rules 自动清理过期图片 |
| 4.4 | 前端部署 | 已有服务器 |

### Phase 5：商业化（P2，后续迭代）

| 序号 | 任务 | 说明 |
|------|------|------|
| 5.1 | 支付接入 | 虎皮椒等聚合支付，前端开放购买入口 |
| 5.2 | 积分套餐 | 入门包/标准包/专业包 |
| 5.3 | 行为验证码 | 生成前触发验证码 |
| 5.4 | 监控告警 | API 花费告警推送 |

## 五、关键技术选型确认

| 组件 | 选型 | 备注 |
|------|------|------|
| 后端框架 | Go (Gin 或 Fiber) | go 的ging                           |
| 前端框架 | Vue 3 + Vite | 可选 Element Plus / Naive UI 组件库 |
| 数据库 | SQLite (go-sqlite3 或 modernc) | 纯 Go 驱动免 CGO 用 modernc |
| 对象存储 | Cloudflare R2 | S3 兼容 API，Go 用 aws-sdk-go-v2 |
| 前端托管 | Cloudflare Pages | 自己部署 |
| 域名/DNS | Cloudflare | 统一管理 DNS + CDN + R2 |
| 认证方式 | JWT | 无状态，适合前后端分离 |
| 限流 | Go 内存限流（如 golang.org/x/time/rate） | 使用同个redis |
